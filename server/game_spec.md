# キャリアすごろく 仕様書 v1.0

## 1. 概要

### 1-1. ゲーム名
キャリアすごろく（仮）

### 1-2. 目的
オンラインで集まった3〜5名が、すごろく形式の対話を通じて以下を学ぶ。
- キャリア観形成
- 仕事理解
- 自己理解
- 他己理解
- 自分の強みの発見
- 他人の強みを見つけるトレーニング

### 1-3. 実施形式
- オンライン開催
- 音声会話は Zoom / ovice を使用
- ゲーム画面は Web ブラウザで共有または各自表示
- ファシリテーター1名が進行を管理

### 1-4. プレイ人数
- 3〜5名
- ファシリテーターはプレイヤー兼任でも専任でもよい

### 1-5. 所要時間
- 60〜90分

## 2. ゲームの基本構造

### 2-1. 全体フロー
1. 導入（ルール説明、キャリアカード配布）
2. 進行（すごろく、質問回答、深掘り）
3. 振り返り（自分と他者が合いそうな職業を選び、理由を共有）

### 2-2. 終了条件
以下のいずれかで終了する。
- 制限時間に達した
- 1人目がゴールした
- ファシリテーターが手動終了した

## 3. コンポーネント

### 3-1. キャリアカード
- 総数: 100種
- 各プレイヤーに5枚配布
- 全体公開

#### 1枚あたりの項目
- id
- 職業名
- カテゴリ
- 仕事内容
- 必要な能力（3〜5個）
- 向いている性格（3〜5個）

### 3-2. ゲームボード
- 総マス数: 40
- 開始マス1
- ゴールマス1
- その他のマス: 38

### 3-3. サイコロ
- 6面

### 3-4. コマ
- プレイヤーを示す色付きトークン

## 4. マス種別
- question: 質問に答える
- deep_dive: 全員が追加質問する
- praise: 全員が強みフィードバックをする
- ask_other: 誰かに質問する
- career_event: キャリアイベントを実行する
- draw_career: キャリアカードを1枚引く
- goal: ゴール

## 5. 1ターンの流れ
1. 現在の手番プレイヤーを表示
2. 手番プレイヤーがサイコロを振る
3. 出目の分だけ進む
4. 止まったマスの内容を表示
5. 対応する対話を行う
6. ファシリテーターが「ターン終了」を押す
7. 次のプレイヤーへ

## 6. 時間目安
- question: 3分
- deep_dive: 5分
- praise: 3分
- ask_other: 3分
- career_event: 3〜4分
- draw_career: 1分

## 7. 振り返りフェーズ
各プレイヤーについて以下を行う。
1. 自分で、手札の中から自分に合いそうな職業を1つ選ぶ
2. 他プレイヤーが、その人に合いそうな職業を1つ選ぶ
3. 理由を共有する
4. 一致点とギャップから気づきを得る

## 8. システム要件

### 8-1. 必須機能
- ルーム作成
- プレイヤー設定
- ファシリテーター設定
- サイコロ
- コマ移動
- マス内容表示
- カード配布
- カード一覧表示
- ターン管理
- 終了条件管理
- 振り返り画面

### 8-2. 任意機能
- タイマー
- ログ表示
- 質問・イベント再抽選
- LocalStorage 保存
- 印刷向けサマリー

## 9. UX方針
- 派手さより、見やすさと進行しやすさを重視
- 日本語 UI
- 研修・ワークショップで扱いやすいこと
- ファシリの強制操作を常に優先

## 10. 実装方針
- フロントエンド完結で始める
- JSON データで内容管理
- 必要になれば後で Firebase / Supabase を追加
