# Codex Prompt 01: Build MVP

このリポジトリに、ファシリテーター付きのオンライン対話ゲーム「キャリアすごろく」のMVPを実装してください。

## ゴール
3〜5人で遊べる、60〜90分想定のオンラインすごろく型キャリア対話ゲームを作る。勝敗はなく、自己理解・他己理解・仕事理解を深めるのが目的。会話はZoom / oviceで行う想定なので、アプリ内ボイスチャットは不要。

## 技術要件
- Next.js 14
- TypeScript
- Tailwind CSS
- 状態管理は Zustand または React Context の軽量なもの
- データは `/data/*.json` を読む構成
- App Router を使う

## MVP要件
1. トップ画面
   - ゲーム概要
   - 新規ルーム作成
   - サンプルセッション開始
2. ルーム画面
   - プレイヤー3〜5人を設定
   - 名前とコマ色を設定
   - ファシリテーターを1人設定
   - 終了条件を選べる（時間制 / 1人目ゴール / ファシリ手動終了）
3. ゲーム画面
   - 40マスの盤面表示
   - 各プレイヤーの現在位置表示
   - 手番表示
   - サイコロを振る
   - コマ移動
   - 止まったマスの内容を右パネルに表示
   - キャリアカード5枚を全員分表示
   - ファシリテーターだけが「次へ」「スキップ」「終了」に触れる
4. 振り返り画面
   - 各プレイヤーが自分に合う職業を1枚選ぶ
   - 他プレイヤーが、その人に合う職業を1枚ずつ選ぶ
   - 理由メモを表示できる
   - 最後に一覧表示

## データ
以下のファイルを読み込んで使うこと
- `data/question_cards_40.json`
- `data/event_spaces_20.json`

## UI方針
- 日本語UI
- 見やすく、研修・ワークショップで使いやすいことを最優先
- 派手なアニメーション不要
- 情報の優先度を明確に

## ほしいコンポーネント例
- Board
- SpaceTile
- PlayerToken
- DicePanel
- TurnPanel
- CardGrid
- PromptPanel
- FacilitatorPanel
- ReflectionPanel

## ロジック要件
- 1ターンでやること: サイコロ → 移動 → マス解決 → ターン終了
- マス種別: question / deep_dive / praise / ask_other / career_event / draw_career / goal
- career_event はイベント一覧からランダムに1件選んで表示
- question は質問カード一覧からランダムに1件選んで表示
- すでに使った質問はなるべく重複しない
- ファシリが強制的に別の質問・イベントへ差し替えられる

## 品質要件
- TypeScriptの型を丁寧に定義
- ロジックは UI からできるだけ分離
- READMEに起動方法を書く
- エラー時の表示を用意する

作業後、変更ファイル一覧と起動手順、未実装項目をまとめてください。
