# キャリアすごろく MVP

ブラウザで遊べる、ファシリテーター進行型のオンライン「キャリアすごろくゲーム」です。  
React + TypeScript のフロントエンドと、Express + Socket.IO のサーバーで構成しています。

## 実装済みMVP

- ファシリ専用ログイン / アカウント管理
- ルーム作成 / 参加
- 3〜5人のプレイヤー管理
- 通常モード / デモモード切替
- 2〜4人の Bot 補完
- ファシリ視点 / プレイヤー視点の切替
- development 環境限定の DeveloperPanel
- 40マスのボード表示
- リアルタイムのターン同期
- サイコロ 1〜6
- コマ移動
- プレイヤー参加型の順番くじ / ファシリによる手番順の入れ替え / 任意マス移動
- マスイベント表示
- 各プレイヤーへ公開キャリアカード5枚を配布
- ファシリテーターによるゲーム開始 / ターン終了
- JSONベースのボード・質問・イベント・キャリアカード管理
- 基本的なゲーム進行ロジックの単体テスト

## キャリアカード設計方針

- キャリアカードは、PDF内の「みんなのハローワークゲーム」の職業一覧を参考に、105職業相当を分析して50職業に整理しました
- `IT / ビジネス / クリエイティブ / 教育 / 医療 / サービス / ものづくり / 公共 / メディア / 自由な働き方` の 10 カテゴリ × 5 職種で構成しています
- プレイヤーが理解しやすい職業を優先し、ゲームの目的である `キャリア観形成` `自己理解` `他己理解` `仕事理解` につながるよう整理しています
- 似た職業や専門領域が近い職業は、ゲーム中に選びやすい粒度へ統合しています
- 他プレイヤーが「この人に合いそう」と選びやすいように、各カードは仕事内容・必要スキル・向いているタイプをシンプルに整理しています
- カード項目は `id / title / category / description / skills / personality` です

## ディレクトリ概要

- `server/` : Express + Socket.IO サーバー
- `app/` : アプリ本体
- `components/` : Lobby / GameBoard / PlayerPanel / EventModal
- `hooks/` : Socket.IO 接続フック
- `lib/` : 型定義とゲーム進行ロジック
- `data/` : ボードやカードのJSONデータ
- `tests/` : ゲームロジックのテスト

## セットアップ

Node.js 18 以上と npm をインストールしてください。

```bash
npm install
```

## 起動方法

```bash
npm run dev
```

起動後:

- フロントエンド: [http://localhost:5173](http://localhost:5173)
- サーバー: [http://localhost:3001](http://localhost:3001)

## ファシリログイン

- ファシリはロビー画面の `ファシリログイン` からログインしてください
- 初期管理者アカウント
- ログインID: `owner`
- 仮パスワード: `AdminTemp2026!`
- 初回ログイン後は、必ずパスワードを変更してください
- 参加者はログイン不要で、URL とルームIDが分かれば参加できます

## 役割別ページ

- 管理人ページ: `/admin`
- ファシリページ: `/facilitator` または `/facilitator/<URLキー>`
- プレイヤーページ: `/player` または `/player/<URLキー>?room=<ルームID>`
- ルート `/` にアクセスすると、3つの入口リンクが表示されます

## ファシリアカウント発行

- 管理者アカウントでログインすると、ロビーに `ファシリアカウント管理` が表示されます
- ここから新しいファシリの `ログインID / 表示名 / 仮パスワード` を発行できます
- パスワードを忘れた場合も、同じ画面から仮パスワードを再発行できます
- 再発行されたファシリは、次回ログイン後にパスワード変更してください
- 管理者画面ではファシリごとの `最終ログイン日時` も確認できます
- 各ファシリには `Primary URLキー / Backup URLキー` の2本を発行します
- 管理者画面から `Primary / Backup` それぞれ個別に再発行できます
- ローカル開発では `data/facilitator_accounts.json` を使います

## Render公開

- 1サービス構成で公開できます
- `npm run build` で Vite の本番フロント資産を `dist/` に生成します
- `npm run start` で Express + Socket.IO サーバーが起動し、`dist/` をそのまま配信します
- 本番ではクライアントの WebSocket 接続先は `window.location.origin` ベースで自動決定されます

Render の設定値:

- Build Command: `npm install && npm run build`
- Start Command: `npm run start`
- Persistent Disk を使う場合は、マウント先を `/var/data` にしてください
- 必要なら環境変数 `FACILITATOR_ACCOUNTS_PATH` で保存先 JSON を明示できます
- 本番ではデフォルトで `/var/data/facilitator_accounts.json` を使います
- `/var/data` に永続ディスクを付けていない場合、再デプロイや再起動でファシリアカウント情報が失われることがあります

## 遊び方

1. 1人目がルームIDと名前を入れてルームを作成します
2. 他の参加者が同じルームIDで参加します
3. 通常モードは 3〜5 人集まったら、デモモードは 1 人でも、ファシリテーターが「ゲーム開始」を押します
4. 手番プレイヤーがサイコロを振ります
5. 止まったマスに応じて EventModal に質問やイベントが表示されます
6. ファシリテーターが対話を進行し、「ターン終了」で次の人へ進めます
7. 終了したいタイミングで、ファシリテーターが `ゲーム終了` を押すと全員に終了画面が表示されます

## デモモード

- ロビーで「デモモード」を選ぶと、1人でも開始できます
- Bot 人数は 2〜4 人から選べます
- Bot 名は `Bot_A`, `Bot_B`, `Bot_C`, `Bot_D` です
- Bot は自分のターンになると自動でサイコロを振り、イベント表示後に自動でターン終了します
- Bot の回答は固定ダミー文言をログへ出します

## 強みカード

- `data/strength_cards.json` の強みカードをゲーム画面から配れます
- 各プレイヤーは、他のプレイヤーに「この人っぽい」と思う強みカードを渡せます
- 配布済みカードは各プレイヤーパネルに表示され、同じカードは同一ルーム内で重複配布されません
- 配布履歴では「誰が誰に渡したか」を確認でき、渡した本人またはファシリテーターは取り消しもできます

## ルール確認と強み一覧

- 画面右上の `ルール` ボタンから、その場でゲームの進め方を確認できます
- `強みカード一覧` ボタンから、配布に使う強みカードをいつでも一覧表示できます
- 常時表示ではなく補助パネル形式なので、ゲーム画面を広く保ったまま参照できます

## ロール切替

- `RoleSwitcher` はデモモードでのみ表示されます
- デモモードでは `ファシリ視点` と `プレイヤー視点` を切り替えられます
- プレイヤー視点では、表示対象プレイヤーを切り替えてカードや進行ログを確認できます
- 通常モードでは、ファシリテーターはファシリ画面、プレイヤーはプレイヤー画面が自動で表示されます

## 進行コントロール

- ファシリテーターは通常画面から、プレイヤーの手番順を手動で入れ替えられます
- ゲーム開始前に、プレイヤーは `順番を引く` ボタンから順番くじを引けます
- 全員が引き終わると、その結果で手番順が自動決定されます
- 遅刻参加や再合流のときは、任意プレイヤーを好きなマスへ移動できます

## 再接続

- ファシリテーターやプレイヤーがリロードしても、30秒以内なら同じルームへ自動再接続します
- その間はルームをすぐ閉じず、接続が戻るのを待つ挙動です
- 30秒を超えて再接続がない場合は、その参加者は退出扱いになります

## ルームIDの扱い

- 同じルームIDがすでに使用中の場合、後から作ろうとした側はエラーになります
- 自動で別のルームIDへ変更する挙動は行いません
- 同時利用時は、ファシリ同士で事前にルームIDを分けてください

## DeveloperPanel

- `development` 環境でのみ表示されます
- 次の操作ができます
- Bot の追加 / 削除
- 任意プレイヤーを現在ターンに設定
- 任意のダイス目で進行
- 任意のマスへ移動
- 任意イベントモーダルの表示
- キャリアカードの再配布
- 現在のゲーム状態 JSON の表示

## 盤面JSONの反映仕様

- `data/board_layout_40.json` は、サーバー起動時の固定読込ではなく、`ルーム作成時に毎回` 最新ファイルを読み込みます
- そのため、盤面JSONを編集したあとは `新規ルームを作成` すれば最新盤面が反映されます
- 既存ルームは作成時点の盤面を保持するため、途中で盤面JSONを変えても既存ルームには反映されません
- 開発中は `BoardDebugPanel` で `roomId / boardVersion / 31〜39 の type` を確認できます
- サーバー側でもルーム作成時に `31〜39` の `type` と `boardVersion` をコンソールへ出力します

## 開発中の確認方法

1. `data/board_layout_40.json` を編集します
2. ブラウザで新しいルームを作成します
3. サーバーのターミナルに出る `board-debug` ログを確認します
4. ブラウザコンソールの `board-debug` ログを確認します
5. 画面内の `BoardDebugPanel` で `boardVersion` と `31〜39` の並びを確認します
6. サーバー・クライアント・画面表示の3つが一致していれば、最新盤面が反映されています
